# Smart Suggestions v4.1.1 — Hardening Round

> Executed inline (small debt round; one review gate before merge). Items and
> approaches originate from the v4.0/v4.1 review findings recorded in the
> specs' deferred lists.

**Goal:** Close the four deferred debts: failed-autorun undo safety, temporal
midnight wrap, renamed switch_as_x mirror dedup, and the flagged test gaps.

## Items

### 1. Activity `success` column (undo safety)
- `activity` table gains `success INTEGER NOT NULL DEFAULT 1`; migration in
  `PatternLedger.init()` via `PRAGMA table_info` check + `ALTER TABLE`.
- `add_activity(ts, sig, act_entity, act_action, success=True)`.
- `execute_autorun` passes the service-call result.
- `_undo`: fire the reverse service ONLY when `entry["success"]`; always mark
  undone + demote (guarded).
- `build_activity_payload` emits `success`; card + panel activity rows append
  "· failed" to the meta line when falsy.
- Tests: ledger roundtrip with success=False; undo-of-failed does not call
  the reverse service (action-handler stub test).

### 2. Temporal midnight wrap
- `ContextMatcher`: minute distance becomes circular:
  `diff = abs(m_now - m_pat); diff = min(diff, 1440 - diff)`.
- `TemporalMiner._find_densest_cluster`: extend the sorted minute axis with
  `+1440` copies, slide the window over the extended axis with window size
  capped at n original points, report `center % 1440`.
- Tests: matcher matches a 23:55 pattern at 00:10; miner clusters events
  straddling midnight (5× 23:55 + 5× 00:05 → one cluster centered ~0:00,
  occurrences 10).

### 3. Renamed mirror dedup
- `HAClient.get_device_id(entity_id) -> str | None` via POST /template
  `{{ device_id('<entity>') | to_json }}`.
- `_dedup_mirrors` becomes async `_dedup_mirrors(candidates, ha)`: normalize
  object_id by stripping a trailing `_<digits>` suffix to FIND light/switch
  candidate pairs with identical (action, miner, details identity); collapse
  only when both entities resolve to the same non-null device_id; prefer the
  `light.` entity. Existing same-object-id dedup keeps working without
  template calls.
- Test: pure normalization/pairing helper unit-tested; device confirmation
  covered via stub HA in the wiring tests.

### 4. Coverage debts
- `test_publisher_contract.py`: sequence-row payload resolves act target from
  details_json (act_entity/act_action + can_promote against target domain,
  incl. a lock-target case asserting can_promote False).
- `test_pattern_ledger.py`: `set_lifecycle` expected-guard (returns False and
  leaves lifecycle unchanged on mismatch).
- New `tests/test_action_handler.py` (stubs): undo idempotency; undo of
  failed autorun demotes without reverse call; promote rejected when
  ineligible; demote guard passes expected=autopilot.
